BasketCase

BasketCase is a (Ruby) script that encapsulates the Rational ClearCase command-line interface, "cleartool", making it (slightly) more comfortable for developers more used to non-locking version-control systems such as CVS or Subversion.

Features

BasketCase can help you:

  • List modified elements.
  • Update a snapshot view, with automatic merge of changes to checked-out elements.
  • Check-out (unreserved) and check-in elements.
  • Undo a check-out, reverting to the base version.
  • Add, remove and rename elements.
  • Display differences for modified elements.
  • Display change-logs and version-trees.

All of the above will seem rather banal to CVS/Subversion users; that's the point - BasketCase is intended to make ClearCase usage intuitive and straightforward, ie. boring.

Getting BasketCase

You can grab the latest version of basketcase.rb directly from RubyForge SVN.

Why BasketCase?

In May 2006, Mike Williams was stuck on a client project which (despite the team's wishes) was stuck using ClearCase as it's version-control system. He wrote BasketCase in frustration.

The team was attempting to use Agile practices such as collective code ownership, refactoring and continuous-integration, and ClearCase was in the way:

  • Depending on the setup, ClearCase likes to do "reserved" check-outs of elements, preventing collective code ownership.
  • When add, removing or moving elements, ClearCase will sometimes apply the change to the repository immediately, without waiting for a "commit".
  • When updating, ClearCase will not attempt to merge other developers' changes to elements you have checked-out; this can leave your view in an inconsistent state.
  • Performing an automatic merge from the command-line requires an unwieldy, obscure command.
  • There is no easy way to do a bulk-commit.
